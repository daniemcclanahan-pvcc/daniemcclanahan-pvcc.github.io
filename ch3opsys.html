Operating Systems Terminology
TERMS DEFINITIONS
Operating System The Software that controls the overall operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to execute the programs requested.
Job Queue A storage organization in which jobs are ordered in first-in, first-out (abbreviated FIFO and pronounced “FI-foe”) fashion. That is, the jobs are removed from the queue in the order in which they arrived
FIFO "first-in, first-out" order used in job queues
Interactive Processing A program being executed to carry on a dialogue with the user through remote terminals. Successful interactive processing is that the actions of the computer be sufficiently fast to coordinate with the needs of the user rather than forcing the user to conform to the machine's timetable.
Time-Sharing Operating systems that provide service to multiple users at the same time
Multiprogramming A technique for time-sharing in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval.
Application Software Applications of programs for performing tasks particular to the machine's utilization. In other words, what is commonly thought of as user installed programs that is not a part of the base system or OS. (database systems, desktop publishing systems, accounting systems, program development software, and games).
System Software The program that performs tasks that are common to computer systems in general.
Utility Software A category of system software that is fundamental to computer installations but not included in the operating system. In a sense, utility software consists of software units that extend (or perhaps customize) the capabilities of the operating system.
Graphical User Interface (GUI) An interface used to communicate with its users in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
Shell Older user interfaces that communicated with users through textual messages using a keyboard and monitor screens.
Kernel An internal part of the operating system that contains software components that perform the very basic functions required by the computer installation.
Device Driver Software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine (such as a printer, disk drive, or monitor), and translates generic requests into the more technical steps required by the device assigned to that driver.
File Manager A component of an operating system's kernel that maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files.
Memory Manager A component of an operating system's kernel which is charged with the task of coordinating the machine's use of main memory, and must find and assign memory space for programs to ensure that the actions of each program are restricted to its allotted space.
Virtual Memory An illusion of additional memory space created by the memory manager by rotating programs and data back and forth between main memory and mass storage.
Boot Strapping A procedure that is performed by a computer each time it is turned on that starts the operating system by tranferring the operating system from mass storage into main memory.
ROM "read-only memory." Memory that uses special nonvolatile memory cells, because its contents can be read but not altered, that store a small portion of a computer's main memory where the CPU expects to find its initial program so the CPU's program counter starts with a particular predeterminded address each time the CPU is turned on.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another. Analysis of deadlock has revealed that it cannot occur unless all three of the following conditions are satisfied:

1.There is competition for nonshareable resources.

2. The resources are requested on a partial basis; that is, having received some resources, a process will return later to request more.

3. Once a resource has been allocated, it cannot be forcibly retrieved.
Spooling A technique of holding data for output at a later but more convenient time that is for granting several processes access to a common resource.
Super User (Or Administrator) An Account that gains highly privileged access to the operating system by identifying him- or herself - as the administrator (usually by name and password) during the login procedure. Once this contact is established, the administrator can alter settings within the operating system, modify critical software packages, adjust the privileges granted to other users, and perform a variety of other maintenance activities that are denied normal users.
Multi-Factor Authentication A security method that requires more than one piece of evidence to gain access (such as special access code, security questions, or biometric credentials) to try and solve the pervasive problem of humans using weak password to secure their computer accounts.A security method that requires more than one piece of evidence to gain access (such as special access code, security questions, or biometric credentials) to try and solve the pervasive problem of humans using weak password to secure their computer accounts.
