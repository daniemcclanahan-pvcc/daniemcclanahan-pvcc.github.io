Chapter 6 Programming Languages
TERMS DEFINITIONS
Machine Language An encoding system for computer programs that consists of sequences of instructions that are encoded as numeric digits.
Debugging The task of locating and correcting errors in a machine language
Assembler A program developed to convert mnemonic expressions into machine language instructions. Thus, rather than being forced to develop a program directly in machine language, a human could develop a program in mnemonic form and then have it converted into machine language by means of an assembler.
Assembly Language A mnemonic system for representing programs.
Translator A program that could translate high-level primitives into machine-language programs.
Compiler Translator programs that is similar to the second-generation assemblers, except that it often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter An alternative program to translators, as another means of implementing third-generation languages, that executed the instructions as they were translated instead of recording the translated version for future use. That is, rather than producing a machine-language copy of a program that would be executed later, an interpreter actually executed a program from its high-level form.
Cross-Platform Software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.
Natural Language The language that people speak and write by that evolved over time without formal grammatical analysis.
Formal Language The language that computers use (aka programming languages) that are precisely defined by grammars.
Machine Independence The ability for code to be written in a high level programming language without knowing details of the physical machine arcitechure for which it will run (code portability).
High-Level Programming Language Powerful and complex programming language that can leverage millions of lines of library code, invoke built-in operating system functions, or initiate complex interactions with networks and databases. As a result, higher-level programming languages make computer programs both easier for human authors to write, and easier for subsequent readers to understand.
Object Oriented Programming Language A programming process that views a software system as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand.
Scripting Language The collection of languages that are used to develop complex programs.
Data Types:
-Integer A way to encode data with numeric data consisting of whole numbers, probably stored using two's complement notation. Operations that can be performed on integer data include the traditional arithmetic operations and comparisons of relative size, such as determining whether one value is greater than another.
-Float A way to encode data with numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
-String A way to encode data which includes character and Boolean. The type character refers to data consisting of symbols, probably stored using ASCII or Unicode. Operations performed on such data include comparisons such as determining whether one symbol occurs before another in alphabetical order.
Primitive Data Type The data types that are included as primitives in a programming language, such as int for integer and char for character.
Data Structure The conceptual shape or arrangement of data. For example, text is normally viewed as a long string of characters whereas sales records might be envisioned as a rectangular table of numeric values, where each row represents the sales made by a particular employee and each column represents the sales made on a particular day.
Pythong Collection Data Structures:
-List A data structure that are used to store multiple items in a single variable. Lists are one of 4 built-in data types in Python used to store collections of data. Lists are created using square brackets: "mylist = ["apple", "banana", "cherry"]"
-Tuple A data structure which is ordered and unchangeable. Tuples are written with round brackets: "mytuple = ("apple", "banana", "cherry")"
-Set A data structure which is unordered, unchangeable, and unindexed. Sets are written with curly brackets: "myset = {"apple", "banana", "cherry"}"
-Dictionary data structure which is ordered*, changeable and do not allow duplicates. Dictionaries are written with curly brackets, and have keys and values: thisdict = {"brand": "Ford", "model": "Mustang", "year": 1964}
Function A program that is viewed as an entity that accepts inputs and produces outputs.
Local Variable A variable declared within a function, meaning that it can be referenced only within that function. This eliminates any confusion that might occur if two functions, written independently, happen to use variables of the same name.
Global Variable Variables whose scopes are not restricted to a particular part of a program.
Source Program In the process of converting a program from one language to another, the program in its original form.
Object Program In the process of converting a program from one language to another, the translated version on the program.
